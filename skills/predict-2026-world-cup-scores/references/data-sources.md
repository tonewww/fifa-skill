# Data Sources

## Source Priority

Use this hierarchy for 2026 FIFA World Cup facts:

1. FIFA official tournament pages, match centre, squad lists, match reports, disciplinary reports, and ranking pages.
2. Confederation and national team official announcements for injuries, replacements, suspensions, and lineups.
3. Established statistical providers for advanced metrics such as xG, shots, pressing, player minutes, market value, and club form. Record provider, URL, retrieval time, and license notes.
4. Reputable reporting for injuries and expected lineups when official confirmation is not available.
5. Analyst judgment only when marked as such in notes and encoded in style profiles or matchup adjustments, not in factual tables.

Do not mix source-backed facts and model opinions in the same field.

- FIFA men's ranking: `https://inside.fifa.com/fifa-world-ranking/men`

As of 2026-06-20, the squad-list PDF URL is live and should be refreshed before use. Treat its `Last-Modified` header as useful provenance.

For rankings, keep two distinct concepts:

- Official FIFA/Coca-Cola Men's World Ranking publication rows in `fifa_rankings` with `ranking_type=official`.
- Any live ranking page/API data in `fifa_rankings` with `ranking_type=live`.

Do not mix live ranking estimates into `teams.fifa_rank` unless the downstream analysis explicitly wants a live-ranking scenario.

## Freshness Windows

Use these maximum ages before refreshing:

- Official lineups: 15 minutes before kickoff until kickoff.
- Injuries, suspensions, withdrawals: 12 hours during matchdays, 24 hours otherwise.
- Fixtures and match status: 30 minutes during live tournament windows.
- Match results and reports: 2 hours after final whistle.
- Final squad lists: 24 hours during the tournament.
- FIFA ranking: 45 days.
- Historical results and player ratings: 7 days unless the provider updates more frequently.
- Style profiles and analyst matchup notes: 7 days, or immediately after a tactical/system change.

If data is stale and cannot be refreshed, say so and increase uncertainty in the prediction.

## Ingestion Flow

1. Archive raw source files or pages in a project-local `data/raw/YYYY-MM-DD/` directory.
2. Convert raw sources to normalized CSVs matching `references/database-schema.md`.
3. Import CSVs with `scripts/import_csv.py`.
4. Run `scripts/validate_database.py --strict`.
5. Rebuild team strength snapshots after every material import.

Do not overwrite raw files without preserving retrieval time. Use source IDs consistently across CSV rows.

## Suggested Normalized CSVs

- `sources.csv`: every source used by the database.
- `teams.csv`: all 48 teams, FIFA code, group, host status, coach, rank, points.
- `players.csv`: official squads plus player-level ratings, expected minutes, and availability.
- `fifa_rankings.csv`: official ranking date, FIFA code, rank, points, prior rank/points.
- `fixtures.csv`: full tournament fixture list and scores when known.
- `team_results.csv`: recent national team results, preferably with xG and opponent quality.
- `player_ratings.csv`: licensed/provider or manually curated player scores.
- `injuries.csv`: injury, suspension, return-date, availability, and impact.
- `lineups.csv` and `lineup_players.csv`: expected/confirmed/official lineups and role weights.
- `tactical_plans.csv`: formation and opponent-specific tactical plan.
- `formation_matchup_stats.csv`: historical formation-pair aggregates, usually generated by script.
- `team_style_profiles.csv`: numeric tactical profile and situational factors.
- `matchup_adjustments.csv`: specific style-counter or personnel notes for one matchup.

## Data Quality Rules

- Require 48 teams before tournament-wide predictions.
- Prefer 26-player final squads when official squads are confirmed.
- Mark player `status` as `available`, `doubtful`, `limited`, `suspended`, `out`, or `withdrawn`.
- Use `minutes_expected` to encode lineup probability and role importance. Starters often sit in the 60-95 range, rotation players 15-45, deep reserves 0-15.
- Keep all ratings on a 0-100 scale.
- For ambiguous names, use stable IDs: team FIFA code plus slugified player name is acceptable until provider IDs are available.
- Store every manual assumption in `notes` or `rationale`.

## Enhancement Import Flow

Use this order when adding the requested enhancement layers:

1. Import `fifa_rankings.csv` with `ingest_fifa_rankings.py --sync-teams`.
2. Import recent results into `team_results`; include xG where licensing permits.
3. Import `player_ratings`, `injuries`, `lineups`, `lineup_players`, and `tactical_plans`.
4. Run `apply_enhancements.py`.
5. Rebuild strength snapshots.
6. Run `analyze_formation_matchups.py` after completed fixtures with usable formations exist.
7. Run `backtest_model.py` and `optimize_model_parameters.py --write-best` after enough completed fixtures exist.

## Official Squad PDF Handling

Preferred extraction path:

1. Download the PDF with `curl -L -o data/raw/YYYY-MM-DD/SquadLists-English.pdf URL`.
2. Extract text with `pdftotext`, `pypdf`, `pdfplumber`, or another available PDF parser.
3. Check that every team has expected player count and that columns such as shirt number, position, club, and date of birth survive extraction.
4. Reconcile extracted team names to `teams.team_id`.
5. Import to `players.csv`.

If the PDF is image-heavy or extraction fails, use OCR or manually curate the normalized CSV. Record the method in the `sources.notes` field.

## Recommended External Metrics

The database schema allows external metrics without requiring one paid provider. Useful additions:

- FIFA ranking points for global baseline.
- Elo or SPI-style team ratings for opponent-adjusted form.
- xG for recent international matches.
- Player club minutes, player ratings, market values, injury status, and role.
- Set-piece goal rates, aerial duel rates, pressing intensity, build-up turnovers, defensive-line height, crossing volume, and goalkeeper shot-stopping.
- Betting market odds can be used as a calibration signal only if legal and appropriate; do not present predictions as betting recommendations.

When a source has licensing restrictions, keep only allowed derived values or require the user to provide the data.

Provider examples are intentionally not hard-coded in scripts. For paid or restricted data, import only fields the user is allowed to store locally, keep provider and URL provenance in `sources`, and avoid redistributing raw tables.

## Public Historical Results

For backtesting pressure tests, `import_historical_results.py` can import public men's international results from:

- `martj42/international_results`: `https://github.com/martj42/international_results`
- Raw CSV default: `https://raw.githubusercontent.com/martj42/international_results/master/results.csv`

Use this source to populate recent `team_results` and completed `fixtures` when official tournament fixtures are not yet complete. Limitations:

- It usually has scorelines and venues, but not xG, shots, lineups, injuries, or tactical details.
- It is suitable for calibration and sanity checks, not a substitute for strict archived pre-match data.
- Only rows with integer final scores are imported; future `NA` score rows are skipped.
- `fixtures` are created only when both teams map to current 2026 participants, while `team_results` are created for any mapped 2026 participant team.

## Exact-Score Odds JSON

When the user provides an exact-score odds JSON, keep it outside the canonical factual tables unless it needs archival. Use `analyze_score_odds_parlay.py` for ad hoc analysis.

Expected shape:

```json
[
  {
    "home_team": "荷兰",
    "away_team": "瑞典",
    "odds": {
      "home_win": {"1:0": 8.0},
      "draw": {"1:1": 6.8},
      "away_win": {"0:1": 16.0}
    }
  }
]
```

Use odds-implied probabilities as a market reference. If the local database lacks recent xG, injuries, lineups, or tactical plans, weight market probabilities higher than the raw model. Always state that parlay output is analytical and not betting advice.
